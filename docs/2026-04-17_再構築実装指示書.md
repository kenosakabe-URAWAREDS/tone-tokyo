## 1. 背景・全体方針

### 1.1 この再構築の目的

TONE TOKYOは、株式会社KAKEHASHIが運営する東京発の英語キュレーターメディアです。2026年4月上旬にPhase 0（基盤構築）を完了し、現在Phase 1（コンテンツシード：30-50記事投入）直前の状態にあります。

Phase 1開始直前の4月17日、運営者の刑部健太郎さんとClaude（Anthropic）の戦略議論により、以下が判明しました：

- **現状のAIパイプラインは動いているが、生成品質が「AI臭い」レベルにとどまっている**
- **AEO基盤（JSON-LD、Answer Block、FAQ等）が実質未実装**
- **編集フローが形骸化している**（`[NEEDS VERIFICATION]`マーカーが本番に残留）
- **Blues Inc.系ブランドの記事扱いルールが未整備**（ステマ規制リスク）
- **運営法人をBlues Inc.から株式会社KAKEHASHIに移管する戦略的決定**

このまま30-50記事投入すると、品質問題が記事数に比例して拡大し、メディアとしての信頼性が構造的に破綻します。よって **Phase 1開始前に、AIパイプライン全体を再設計する** 必要があります。

### 1.2 目標水準

**プロの編集者に見抜かれないAI生成記事を量産する。**

これは極めて高いハードルです。世界のAIメディア実験（BuzzFeed、CNET、Sports Illustrated等）はこの目標で失敗しています。しかし健太郎さんには他メディアにない武器があります：

- 15年のKURO経営で築いた職人・工場との固有の関係
- 富ヶ谷/代々木公園エリア10年以上の生活者視点
- Felix（ベルリン）、Hugo（スペイン）、Shimizuさんとのグローバル一次情報
- THE UNION運営による海外バイヤー・デザイナー接点
- 毎日通う店、本当に使うブランド、実在の人間関係

**AI臭さを消す鍵は、AI技術ではなく、健太郎さんの一次情報の活用です。**

### 1.3 再設計の核心：パイプライン再定義

**旧設計（Phase 0時点）:**
```
AI自動生成(40%) + 健太郎手動(60%) = 100%
```

**新設計:**
```
健太郎の一次情報インプット(5分/日・音声メモ)
  × AIが深掘り逆質問して固有情報を引き出す
  × AIによる骨格化・構成・事実補強
  × AI臭さ検知スコアリング
  = プロ編集者レベルの記事
```

AIは「記事を書く」ではなく「健太郎のインプットを記事に変換する」役割に徹する。

### 1.4 記事タイプ2区分

全記事を2タイプに分類し、品質基準を変える：

| タイプ | 目標 | AI臭さ基準 | 分量 | 用途 |
|---|---|---|---|---|
| **Editorial** | プロ編集者に見抜かれない | 厳格 | 800-2000語 | ピラーの中核。健太郎の一次情報由来 |
| **News** | 情報速度優先 | 緩め（許容） | 200-400語 | 業界ニュース・イベント速報 |

視覚的にも明確に区別する（Editorialは大きなヒーロー、Newsはコンパクトリスト）。

### 1.5 運営法人の変更

**旧:** Blues Inc.（有限会社ブルース）運営
**新:** 株式会社KAKEHASHI運営（Felixと共同運営のTHE UNION法人）

これに伴い：
- Footer / About / Organization JSON-LD / 特商法ページ / プライバシーポリシー全てに「Operated by KAKEHASHI Inc.」明記
- Blues Inc.系ブランド（KURO, VONN, THE BLUE STORE, AIZOME REWEAR, LYNARC等）記事に disclosure 必須

```typescript
// scripts/count-articles.ts
import { createClient } from '@sanity/client'

const client = createClient({
  projectId: 'w757ks40',
  dataset: 'production',
  apiVersion: '2026-04-17',
  useCdn: false,
  token: process.env.SANITY_API_TOKEN
})

async function countArticles() {
  const total = await client.fetch('count(*[_type == "article"])')
  const published = await client.fetch('count(*[_type == "article" && !(_id in path("drafts.**")) && defined(publishedAt)])')
  const draft = await client.fetch('count(*[_type == "article" && _id in path("drafts.**")])')

  const all = await client.fetch('*[_type == "article"]{_id, title, slug, publishedAt, _createdAt}')

  // 重複検出: 同じ title や slug
  const titleMap = new Map()
  const slugMap = new Map()
  const duplicates = { byTitle: [], bySlug: [] }

  all.forEach(a => {
    if (titleMap.has(a.title)) duplicates.byTitle.push({ title: a.title, ids: [titleMap.get(a.title), a._id] })
    titleMap.set(a.title, a._id)

    const slug = a.slug?.current
    if (slug && slugMap.has(slug)) duplicates.bySlug.push({ slug, ids: [slugMap.get(slug), a._id] })
    if (slug) slugMap.set(slug, a._id)
  })

  console.log({ total, published, draft, duplicates })
  console.log('All articles:', JSON.stringify(all, null, 2))
}

countArticles()
```

```typescript
// scripts/migrate-articles.ts
import { createClient } from '@sanity/client'

const client = createClient({
  projectId: 'w757ks40',
  dataset: 'production',
  apiVersion: '2026-04-17',

// 健太郎さんの指示に基づいて editorial/news を判定
const articleTypeMap: Record<string, 'editorial' | 'news'> = {
  'gyoryoku-oku-shibu-a-meiji-era-fishmonger-s-teishoku-counter': 'editorial',
  'sushi-onikai-1-proper-edomae-at-a-fair-price': 'news',
  'sushi-w-delivers-proper-omakase-without-the-manhattan-price-tag': 'news',
  'the-price-of-perfection-at-kioi-cho-mitani-bettei': 'editorial',

  'lynarc-s-hematine-infused-hair-care-creates-salon-results-at-home': 'editorial',
  'the-unchanging-comfort-of-yoshokuya-b-in-ikejiri-ohashi': 'editorial',
  'twenty-years-of-slurping-at-umegaoka-s-katsuya': 'editorial',
  'the-blue-store-tokyo-s-most-understated-cool': 'editorial',
  'okayama-selvedge-denim-weavers': 'editorial',
  // その他の記事は調査結果から追加
}

// ブランド言及マッピング
const brandMentionsMap: Record<string, { brands: string[], level: number }> = {
  'lynarc-s-hematine-infused-hair-care-creates-salon-results-at-home': {
    brands: ['lynarc'],
    level: 3
  },
  'the-blue-store-tokyo-s-most-understated-cool': {
    brands: ['the_blue_store'],
    level: 3
  },
  'okayama-selvedge-denim-weavers': {
    brands: ['kuro'],
    level: 1
  },
  // デフォルトは level 0, brands []
}

async function migrate() {
  const articles = await client.fetch('*[_type == "article"]{_id, slug}')

  for (const article of articles) {
    const slug = article.slug?.current
    if (!slug) continue

    const articleType = articleTypeMap[slug] || 'editorial'
    const brandInfo = brandMentionsMap[slug] || { brands: [], level: 0 }

    const patch = client.patch(article._id)
      .set({
        articleType,
        brandMentions: brandInfo.brands,
        brandMentionLevel: brandInfo.level,
        hasForbiddenMarkers: false
      })

    await patch.commit()
    console.log(`Migrated: ${slug} → ${articleType}, brands: ${brandInfo.brands}, level: ${brandInfo.level}`)
  }

  console.log('Migration complete')
}

migrate().catch(console.error)

## パート5: AI深掘り対話システム

### このパートの目的
健太郎さんの短い一次情報インプット（音声メモや数行のテキスト）に対して、AIが逆質問を重ねて**記事に必要な固有情報を引き出す**対話システムを実装する。これがAI臭さを消す核心。

### 前提条件
- パート4完了・承認済み
- InputTabで「ネタ帳に保存」が動作
- Anthropic API Key（`ANTHROPIC_API_KEY`）が環境変数に設定済み

### 完了条件
- ネタ帳のアイテムをクリックすると、AIとの対話セッションが開始
- AIが健太郎さんに3-5回の深掘り逆質問を実施
- 対話履歴がSanityに保存される
- 対話完了後、蓄積された情報から記事ドラフトを生成
- 生成された記事ドラフトが Sanity の drafts に保存される

#### 5-3. 深掘り対話システムプロンプト設計

```typescript
// lib/dialogue-prompts.ts

export const DIALOGUE_SYSTEM_PROMPT = `
あなたはTONE TOKYO という東京発の英語キュレーターメディアの編集補助AIです。
運営者の健太郎（The Editor）が短いインプットを投げてきた時、
あなたは「プロの編集者に見抜かれないレベルの記事」を作るために必要な
固有情報を引き出す逆質問をします。

## あなたの役割
- 健太郎の短いメモから、記事に必要な「固有性の高い情報」を引き出す
- 1回に3-5個の具体的な質問を投げる（多すぎない）
- 質問は健太郎が「その場でパッと答えられる」レベルの具体性
- 抽象的な質問（「どう感じましたか？」）ではなく、具体的な事実を問う

## 引き出すべき情報の種類

### 1. 固有名詞
- 人名（店主、職人、デザイナー等のフルネーム or 呼び名）
- 店名・ブランド名・会社名
- 地名・住所・最寄り駅

### 2. 具体的な数字
- 創業年・築年数
- 価格（円単位）
- 距離・時間・容量
- 人数・回数

### 3. 具体的エピソード
- いつ行ったか
- 何を注文したか/見たか
- 誰と会話したか・何を話したか
- 失敗談・意外な出来事

### 4. 感覚的ディテール（五感）
- 匂い・音・温度・光・手触り
- 特徴的な表情・仕草
- 店の雰囲気の具体的描写

### 5. 個人的な評価・偏見
- 健太郎の率直な好み（「実はXが好きではない」）
- 他との比較での明確な位置づけ
- 何年通っているか、どのくらいの関係か

## 質問の出し方

健太郎からの最初のインプット例:
「今日Collect Mills行った」

悪い質問例（抽象的・AIっぽい）:
- 「どんな印象でしたか？」
- 「工場の雰囲気について教えてください」
- 「どんな学びがありましたか？」

良い質問例（具体的・記者的）:
- 「Nakagawaさん元気でしたか？前回体調崩していたと仰ってましたが」
- 「どの機械で織っているところを見ましたか？型番覚えてます？」
- 「今日何色のデニム織ってた？何オンス？」
- 「帰りに食べた飯屋どこ？」
- 「工場の匂い、前回と変わった感じありました？」

## 対話の進行

1. 1回目: 最も重要な固有情報を3-5問聞く
2. 2回目: 1回目の回答から深掘り（追加の人物、数字、エピソード）
3. 3回目: 感覚的ディテールと個人的評価を引き出す
4. 4回目以降: 必要に応じて追加質問
5. 5回目以降または十分な情報が集まったら: 「十分な情報が集まりました。記事ドラフトを生成しますか？」と聞く

## 重要ルール

- 必ず日本語で質問する（健太郎が日本語で返答するため）
- 1回の応答で3-5問に留める（健太郎の認知負荷を下げる）
- 箇条書きで見やすく質問を列挙
- 健太郎が「知らない」「覚えていない」と答えたら、その情報は記事に入れない方針を伝える
- 健太郎が疲れている様子なら、質問を減らす
`

#### 5-4. 対話APIエンドポイント

```typescript
// app/api/dialogue/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { anthropic, CLAUDE_MODEL } from '@/lib/anthropic'
import { sanityWriteClient } from '@/lib/sanity'
import {
  DIALOGUE_SYSTEM_PROMPT,
  buildInitialQuestionPrompt,
  buildFollowUpPrompt,
  buildFactExtractionPrompt
} from '@/lib/dialogue-prompts'

export async function POST(req: NextRequest) {
  const { ideaId, action, userMessage } = await req.json()

  const idea = await sanityWriteClient.fetch(
    `*[_type == "idea" && _id == $id][0]`,
    { id: ideaId }
  )

  if (!idea) {
    return NextResponse.json({ error: 'Idea not found' }, { status: 404 })
  }

  const history = idea.dialogueHistory || []

  // Actionに応じて処理
  if (action === 'initial_question') {
    // 最初の質問生成
    const prompt = buildInitialQuestionPrompt(idea.rawInput)

    const response = await anthropic.messages.create({
      model: CLAUDE_MODEL,
      max_tokens: 1024,
      system: DIALOGUE_SYSTEM_PROMPT,
      messages: [{ role: 'user', content: prompt }]
    })

    const aiMessage = response.content[0].type === 'text' ? response.content[0].text : ''

    // Sanityに保存
    const newHistory = [
      ...history,
      { role: 'ai', message: aiMessage, timestamp: new Date().toISOString() }
    ]

    await sanityWriteClient.patch(ideaId)
      .set({ dialogueHistory: newHistory, status: 'in_dialogue' })
      .commit()

    return NextResponse.json({ aiMessage, history: newHistory })
  }

  if (action === 'follow_up') {
    // 健太郎の回答を保存
    const updatedHistory = [
      ...history,
      { role: 'editor', message: userMessage, timestamp: new Date().toISOString() }
    ]

    // AIからの次の質問を生成
    const prompt = buildFollowUpPrompt(updatedHistory)

    const response = await anthropic.messages.create({
      model: CLAUDE_MODEL,
      max_tokens: 1024,
      system: DIALOGUE_SYSTEM_PROMPT,
      messages: [{ role: 'user', content: prompt }]
    })

    const aiMessage = response.content[0].type === 'text' ? response.content[0].text : ''

    const finalHistory = [
      ...updatedHistory,
      { role: 'ai', message: aiMessage, timestamp: new Date().toISOString() }
    ]

    await sanityWriteClient.patch(ideaId)
      .set({ dialogueHistory: finalHistory })
      .commit()

    return NextResponse.json({ aiMessage, history: finalHistory })
  }

  if (action === 'extract_facts') {
    // 対話終了 → 固有情報抽出
    const prompt = buildFactExtractionPrompt(history)

    const response = await anthropic.messages.create({
      model: CLAUDE_MODEL,
      max_tokens: 2048,
      messages: [{ role: 'user', content: prompt }]
    })

    const jsonText = response.content[0].type === 'text' ? response.content[0].text : '{}'

    // JSON抽出（```jsonフェンスがあれば除去）
    const cleanJson = jsonText.replace(/```json|```/g, '').trim()
    const extractedFacts = JSON.parse(cleanJson)

import { NextRequest, NextResponse } from 'next/server'
import { anthropic, CLAUDE_MODEL } from '@/lib/anthropic'
import { sanityWriteClient } from '@/lib/sanity'

const SCORING_SYSTEM_PROMPT = `
あなたはメディア記事の「AI臭さ」を検知する専門家です。
プロの編集者が記事を読んで「これはAI生成だ」と見抜くかどうかを、
5つの指標で評価します。
`

const SCORING_PROMPT_TEMPLATE = `
以下の英語記事を評価してください。記事タイプは {articleType} です。

---記事本文---
{articleBody}
---

5つの指標で0-100のスコアを付けてください:

## 1. Proper Noun Density (固有名詞密度)
- 人名（フルネーム or 呼び名）、店名、地名、ブランド名、商品名の出現数
- 500語あたりの固有名詞数で計測
- 基準: {pnDensityThreshold}個/500語 以上で満点100

## 2. Specific Numbers (具体的数字)
- 年号、価格、距離、時間、人数、容量等の具体的数値
- 「created in 1958」「priced at ¥3,400」「30 minutes walk」等
- 「many」「several」「a few」といった曖昧表現は逆に減点
- 基準: {numbersThreshold}個以上で満点100

## 3. Sensory Concreteness (感覚描写の具体性)
- 匂い、音、温度、光、手触りの具体的描写
- 一般的形容詞（"warm glow"、"rich aroma"）は減点
- 個別具体的な描写（"oil spattering onto the white apron"、"the miso smelled like burning sugar"）は加点

## 4. Personal Bias (個人的偏見)
- 「I don't like X」「This is overrated」「My preferred...」等の個人的判断
- 「balanced」「overall positive」の中立評価は減点
- 基準: 最低1箇所の明確な偏見表現で合格

## 5. Rhythm Variation (リズム変動)
- 文の長さの変動が大きいほど高得点
- 「短い一文。その後に長い複雑な文が続く。」というパターン
- 均一な長さの文が続くとAI生成っぽい（減点）

## 総合判定

- editorialモード: 全指標 75以上で合格
- newsモード: Proper Noun Density と Specific Numbers のみ 60以上で合格（他は不問）

以下JSONで出力してください:

{
  "properNounDensity": {
    "score": 0-100,
    "count": 実際の固有名詞数,
    "per500Words": 500語あたりの計算値,
    "examples": ["見つけた固有名詞の例"]
  },
  "specificNumbers": {
    "score": 0-100,
    "count": 実際の具体的数字数,
    "examples": ["見つけた具体的数字の例"]
  },
  "sensoryConcreteness": {
    "score": 0-100,
    "concreteExamples": ["具体的な描写の例"],
    "genericExamples": ["一般的な形容詞の例"]
  },
  "personalBias": {
    "score": 0-100,
    "biasStatements": ["個人的偏見の表現例"]
  },
  "rhythmVariation": {
    "score": 0-100,
    "averageSentenceLength": 平均文長,
    "standardDeviation": 標準偏差
  },
  "overall": 5指標の加重平均（editorial重視またはnews重視）,
  "passed": true | false,
  "feedback": "不合格の場合、改善提案を3つ程度",
  "problematicPhrases": ["AI臭いと判定された具体的なフレーズ"]
}
`

const EDITORIAL_THRESHOLDS = {
  pnDensity: 15, // 500語あたり15個以上
  numbers: 5,
  overall: 75
}

const NEWS_THRESHOLDS = {
  pnDensity: 8,
  numbers: 3,
  overall: 60
}

export async function POST(req: NextRequest) {
  const { articleId } = await req.json()

  const article = await sanityWriteClient.fetch(
    `*[_type == "article" && _id == $id][0]`,
    { id: articleId }
  )

  if (!article) {
    return NextResponse.json({ error: 'Article not found' }, { status: 404 })
  }

  const articleType = article.articleType || 'editorial'
  const thresholds = articleType === 'editorial' ? EDITORIAL_THRESHOLDS : NEWS_THRESHOLDS

  // 本文をプレーンテキストに変換
  const bodyText = (article.body || [])
    .map((block: any) => {
      if (block._type === 'block') {
        return block.children?.map((c: any) => c.text).join(' ') || ''
      }
      return ''
    })
    .join('\n\n')

  const prompt = SCORING_PROMPT_TEMPLATE
    .replace('{articleType}', articleType)
    .replace('{articleBody}', bodyText)
    .replace('{pnDensityThreshold}', thresholds.pnDensity.toString())
    .replace('{numbersThreshold}', thresholds.numbers.toString())

  const response = await anthropic.messages.create({
    model: CLAUDE_MODEL,
    max_tokens: 4096,
    system: SCORING_SYSTEM_PROMPT,
    messages: [{ role: 'user', content: prompt }]
  })

  const jsonText = response.content[0].type === 'text' ? response.content[0].text : '{}'
  const cleanJson = jsonText.replace(/```json|```/g, '').trim()
  const scores = JSON.parse(cleanJson)

  // Sanityに保存
  await sanityWriteClient.patch(articleId)
    .set({
      aiGenerationScore: {
        overall: scores.overall,
        properNounDensity: scores.properNounDensity.score,
        specificNumbers: scores.specificNumbers.score,
        sensoryConcreteness: scores.sensoryConcreteness.score,
        personalBias: scores.personalBias.score,
        rhythmVariation: scores.rhythmVariation.score,
        evaluatedAt: new Date().toISOString(),
        passed: scores.passed
      }
    })
    .commit()

  return NextResponse.json({
    scores,
    thresholds,
    passed: scores.passed
  })
}

      model: CLAUDE_MODEL,
      max_tokens: 1024,
      messages: [{ role: 'user', content: prompt }]
    })

    const aiMessage = response.content[0].type === 'text' ? response.content[0].text : ''

    // ideaの dialogueHistory に追加
    const updatedHistory = [
      ...idea.dialogueHistory,
      { role: 'ai', message: aiMessage, timestamp: new Date().toISOString() }
    ]

    await sanityWriteClient.patch(idea._id)
      .set({
        dialogueHistory: updatedHistory,
        status: 'in_dialogue'
      })
      .commit()

    return NextResponse.json({ aiMessage, ideaId: idea._id })
  }

  if (mode === 'regenerate') {
    // 既存 extractedFacts で記事を再生成
    const generateRes = await fetch(
      `${process.env.NEXT_PUBLIC_SITE_URL}/api/generate-article`,
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ ideaId: idea._id })
      }
    )
    const data = await generateRes.json()
    return NextResponse.json(data)
  }

  return NextResponse.json({ error: 'Invalid mode' }, { status: 400 })
}

function extractBodyText(body: any[]): string {
  return (body || [])
    .map((block: any) => {
      if (block._type === 'block') {
        return block.children?.map((c: any) => c.text).join(' ') || ''
      }
      return ''
    })

export function generateDisclosure(
  brandMentions: string[] | undefined,
  level: number | undefined
): string | null {
  if (!brandMentions || brandMentions.length === 0 || !level || level < 2) return null

  const brands = brandMentions
    .map(b => BRAND_INFO[b])
    .filter(Boolean)

  if (brands.length === 0) return null

  const brandNames = brands.map(b => b.name).join(', ')
  const relationships = brands.map(b => b.relationship).join('; ')

  if (level === 3) {
    // フル disclosure
    return `TONE TOKYO is operated by KAKEHASHI Inc. Coverage of ${brandNames} reflects personal experience of The Editor, who is ${relationships}. This piece is not sponsored content. Editorial decisions are independent.`
  } else {
    // Level 2: 簡易 disclosure
    return `The Editor has professional relationships with ${brandNames}. Coverage here reflects direct experience and is not sponsored.`
  }
}

export async function getRelatedArticles(
  currentArticleId: string,

        url: 'https://www.tone-tokyo.com/logo.png'
      },
      parentOrganization: {
        '@type': 'Organization',
        name: 'KAKEHASHI Inc.'
      }
    },
    mainEntityOfPage: {
      '@type': 'WebPage',
      '@id': `https://www.tone-tokyo.com/article/${article.slug.current}`
    },
    articleSection: article.pillar,
    keywords: article.tags?.join(', '),
    wordCount: estimateWordCount(article.body),
    inLanguage: 'en'
  }

  // FAQPage スキーマ（FAQがあれば別途追加）
  const faqJsonLd = article.faqs?.length > 0 ? {
    '@context': 'https://schema.org',
    '@type': 'FAQPage',
    mainEntity: article.faqs.map((faq: any) => ({
      '@type': 'Question',
      name: faq.question,
      acceptedAnswer: {
        '@type': 'Answer',
        text: faq.answer
      }
    }))
  } : null

  // BreadcrumbList
  const breadcrumbJsonLd = {
    '@context': 'https://schema.org',
    '@type': 'BreadcrumbList',
    itemListElement: [
      {
        '@type': 'ListItem',
        position: 1,
        name: 'Home',
        item: 'https://www.tone-tokyo.com/'
      },
      {
        '@type': 'ListItem',
        position: 2,
        name: article.pillar,
        item: `https://www.tone-tokyo.com/discover?pillar=${article.pillar}`
      },
      {
        '@type': 'ListItem',
        position: 3,
        name: article.title
      }
    ]
  }

  return (
    <>
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
      />
      {faqJsonLd && (
        <script
          type="application/ld+json"
          dangerouslySetInnerHTML={{ __html: JSON.stringify(faqJsonLd) }}
        />
      )}
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(breadcrumbJsonLd) }}
      />
    </>
  )
}

function estimateWordCount(body: any[]): number {
  return (body || [])
    .map((block: any) => {
      if (block._type === 'block') {
        return block.children?.map((c: any) => c.text).join(' ') || ''
      }
      return ''
    })
    .join(' ')
    .split(/\s+/)
    .length
}

## トラブルシューティング

### PowerShell BOM問題

**症状:** Next.js/TypeScriptファイルに BOM が入り、ビルドエラー

**対処:**
```powershell
# ❌ 悪い例
Set-Content -Encoding UTF8 $path $text

# ✅ 良い例
$utf8NoBom = New-Object System.Text.UTF8Encoding($false)
[System.IO.File]::WriteAllText($path, $text, $utf8NoBom)
```

### [slug] ブラケット問題

**症状:** `app\article\[slug]\` のようなパスで PowerShell が失敗

**対処:**
```powershell
# ✅ -LiteralPath フラグ使用
Get-Content -LiteralPath "app\article\[slug]\page.tsx"

# または cmd 経由
cmd /c type "app\article\[slug]\page.tsx"
```

### Sanity GROQ 画像クエリ

**症状:** `heroImage.asset->url` のみでは外部URL（heroImageUrl）が取得できない

**対処:**
```groq
"heroImageUrl": coalesce(heroImage.asset->url, heroImageUrl)
```

### IntersectionObserver useVisible

**症状:** `useState(false)` で初期化すると記事が `opacity:0` のまま

**対処:**
```tsx
// ❌
const [isVisible, setIsVisible] = useState(false)

// ✅
const [isVisible, setIsVisible] = useState(true)
```

### Vercel 4.5MB リクエスト上限

**症状:** 画像アップロード時に 413 エラー

**対処:** Canvas圧縮（max 1200px、JPEG 0.7）を実装。または Sanity 直接アップロード方式に変更。

### マルチライン文字列置換の失敗

**症状:** PowerShellのマルチライン一致が失敗

**フォールバック順:**
1. `cmd /c` 経由
2. `notepad` 直接編集
3. Node.js スクリプトファイル（`_write.js` 等）で書く
